# Skill: grill-with-docs

Use antes de alterar regras editoriais, progressão de envelopes, dificuldade, dicas, guia do facilitador, mapas, documentos de jogador, pipeline de criação ou decisões de produto.

## Procedimento

1. Leia `AGENTS.md` e `docs/LLM_CONTEXT.md`.
2. Leia os docs relacionados à área afetada.
3. Confronte a mudança com as regras atuais.
4. Identifique tensão, risco ou contradição.
5. Proponha a menor mudança coerente.
6. Atualize documentação se a regra mudar.
7. Valide conforme o escopo.

## Docs por escopo

- Narrativa/progressão: `docs/CASE_DESIGN_PIPELINE.md`, `docs/BLUEPRINT_AUTHORING_GUIDE.md`, `docs/DIRETRIZES_EDITORIAIS.md`.
- LLM/autoria: `docs/LLM_OPERATING_MANUAL.md`.
- Mapas: `docs/FLOORPLANS.md`, `docs/VISUAL_LIBRARY_2_0.md`.
- Visual/renderização: `docs/VISUAL_SYSTEM.md`, `docs/PRINTABLES.md`.
- Assinaturas: `docs/SIGNATURES_AND_HANDWRITING.md`.
- Estado/roadmap: `docs/ESTADO_ATUAL.md`, `docs/ROADMAP.md`.

## Perguntas obrigatórias

- Melhora solvabilidade, diversão, clareza ou baseline visual?
- Preserva offline-first?
- Mantém evidência bruta nos documentos de jogador?
- Evita voz do autor, dica óbvia ou gabarito disfarçado?
- Respeita o papel dos envelopes?
- Adiciona complexidade antes do playtest sem evidência?

## Resposta final

Informe:

- skill: `grill-with-docs`;
- docs consultados;
- decisão tomada;
- trade-off identificado;
- arquivos alterados;
- validações executadas;
- pontos não alterados e motivo.
