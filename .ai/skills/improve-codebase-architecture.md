# Skill: improve-codebase-architecture

Use em revisão arquitetural explícita.

## Passos

1. Leia `AGENTS.md`.
2. Leia `docs/LLM_CONTEXT.md`.
3. Mapeie responsabilidades atuais.
4. Liste problemas com evidência.
5. Proponha mudanças pequenas e reversíveis.
6. Defina validação.

## Saída

- skill usada;
- diagnóstico;
- problemas comprovados;
- recomendações por prioridade;
- itens que devem esperar;
- plano de validação.

## Guardrails

- Não criar banco de dados, dashboard, marketplace, multiusuário, Telegram comercial ou IA de imagem por iniciativa própria.
- Não trocar Playwright/pikepdf sem evidência forte.
- Não reescrever o pipeline antes de baseline visual/playtest.
